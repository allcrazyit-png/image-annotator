# UX 改版設計：圖片標注工具直覺化

**日期：** 2026-05-25
**版本：** v1.0
**方向：** 情境感知面板（選項 B）

## 背景與問題

使用者主要在手機與桌機上使用本工具，場景為標注錯誤圖片（箭頭指向、圓圈圈出、文字說明），且常需要在放下標注後回頭修改。

確認的四個痛點：

1. **行動版工具列太難用**：超過 10 個工具橫向排列，需捲動，目前選的工具不明顯。
2. **選取、移動、刪除不直觀**：必須切換到「選取工具」才能點標注，刪除路徑不清晰。
3. **文字無法二次編輯**：放下文字後想修改，只能刪掉重畫。
4. **顏色/粗細作用對象不清**：不確定改的是選取中的標注，還是下一個要畫的。

## 設計目標

- 任何情境下都能直接修改已放下的標注，不需切換工具
- 行動版工具列精簡，常用工具一眼可見
- 顏色與粗細的作用對象永遠清楚標示

## 不在本次範圍

- 縮放（zoom）功能
- 圖層管理
- 多人協作
- 整體視覺設計更換

## 設計細節

### 1. 選取模型

**行為改變：**
- 任何繪圖工具模式下（箭頭、圓圈、文字等），點擊已存在的標注都能選取它，不需先切換到「選取工具」。
- 畫完一個標注後，該標注自動進入選取狀態（藍色虛線框 + 縮放把手）。
- 點空白畫布處解除選取，當前工具保持不變。
- 「選取工具」按鈕仍保留在工具列（桌機側欄、手機「全部工具」tab），供需要純選取模式的使用者使用。

**選取視覺：**
- 選取框：藍色虛線（現有樣式保持不變）
- 四角縮放把手：現有圓形把手保持不變

**懸浮快捷列（Floating Action Bar）：**
- 選取任何標注後，在標注上方出現黑色小列
- 所有標注類型顯示：`✕ 刪除`
- 僅文字（text）類型額外顯示：`✏ 編輯`
- 點擊畫布其他地方後消失

**實作要點：**
- 在 `onDown()` 的最前面，任何工具模式下都先執行 `findHitAnnotation(p)`
- 如果命中，選取該標注並阻止繪圖行為；如果未命中，正常繼續繪圖
- 選取後立即可拖移（不需二次點擊）

### 2. 情境感知面板

**桌機右側面板：**

| 狀態 | 面板顯示內容 |
|------|-------------|
| 無選取 | 標題「下一個標注的顏色」+「下一個標注的粗細」+ 草稿/匯出區 |
| 選取圓圈/矩形/箭頭/手繪/螢光筆 | 標題「選取中：XXX」+ 顏色 + 粗細 + 刪除按鈕 |
| 選取文字 | 標題「選取中：文字」+ 顏色 + 文字大小 + 「✏ 編輯文字」+ 刪除按鈕 |
| 選取數字編號 | 標題「選取中：編號」+ 顏色 + 大小 + 刪除按鈕 |

**手機底部色條（`#mobile-opts`）：**

- 最左側新增標籤文字：無選取時顯示灰色「下一個」，有選取時顯示藍色「選取中」
- 有選取文字類型時，色條右側新增「✏ 編輯」按鈕
- 有選取任何標注時，色條最右側新增「✕ 刪除」按鈕

### 3. 文字二次編輯

- 選取文字標注後，桌機面板出現「✏ 編輯文字內容」按鈕，手機色條出現「✏ 編輯」按鈕
- 點擊任一按鈕：開啟現有的 `#text-modal`，`#text-field` 預填為該標注的 `a.text` 值
- 確認後直接更新 `a.text`，不新增標注
- 桌機另支援**雙擊**文字標注觸發相同流程

### 4. 行動版工具列精簡

現有 `#mobile-bar` 改為加上兩個 tab，tab bar 固定在工具列上方。

**常用工具 tab（預設顯示）：**
上傳、`分隔線`、箭頭、文字、圓圈、編號、`分隔線`、復原、重做、`分隔線`、下載、分享

**全部工具 tab：**
顯示所有現有工具，包含矩形、手繪、螢光筆、馬賽克、選取工具、清除全部、草稿儲存

**Tab 切換行為：**
- 切換 tab 時只切換工具列內容，不影響當前選取的工具
- 記憶上次使用的 tab（`localStorage`）

### 5. 預設值調整

| 參數 | 舊值 | 新值 |
|------|------|------|
| 線條粗細 (`strokeW`) | 3 | 6 |
| 文字大小 (`fontSize`) | 120 | 180 |

已於 2026-05-25 直接套用至 `index.html`。

## 實作注意事項

- `onDown()` 邏輯需改為：先全域偵測點擊命中，再依工具分支
- 懸浮快捷列使用 DOM 元素覆蓋在 `#canvas-wrapper` 上，用 `canvas.getBoundingClientRect()` 定位
- 情境面板切換透過 CSS class（`show`/`hide`）控制，避免頻繁 DOM 重建
- 文字編輯需區分「新增模式」與「編輯模式」，可用 `S.editingIdx` 旗標追蹤
- 手機 tab 狀態存入 `localStorage('annotator-tab')`，初始化時讀取
